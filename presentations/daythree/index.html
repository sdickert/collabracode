What is jQuery?

  • jQuery is a function, like any other JavaScript function.
  • It is "overloaded", i.e., it responds differently to different types of arguments.
  • In all cases, it returns a "jQuery Object" - an object with convenience methods for working with a wrapped array of DOM elements

¡Notice!

We will be using the shortcut variable ($) to indicate use of the jQuery object.

It is important to note this because you can use the jQuery library even with other libraries that use the $ shortcut by calling jQuery.noConflict();

Examples

  • $('selector', [context])
  • $('
    Some HTML
    ')
  • $(element)
  • $()
  • $(function () { ... })

What is a “jQuery Object”?

  • The object returned by invoking jQuery
  • An Array of DOM Elements matching a selector with cross browser methods for traversing and manipulating the elements in the Array
  • Every call to jQuery's traversal/selection methods returns a new jQuery object.

$("#submit")
    .css({'background' : 'blue'})
    .removeAttr('disabled')
    .trigger('click');

Why the $?

  • A $ is a valid JavaScript variable
  • var $ = 'My Name';
alert($);
  • By default, the jQuery object is copied to $
  • $('selector') === $('selector');

Other jQuery Functions

  • Other utility functions are stored within the jQuery object.

  • $.ajax
  • $.fn
  • More then can be listed here

Pass it a “Selector”

A selector is a string with a CSS pattern used for matching elements.

When you pass jQuery a selector, it queries the DOM to return you a set of elements that match the selector.

$("#anElementsID");
$("div.allDivsWithClassFoo");
$("input[name='city']");
$("span");

Pass it DOM Elements

If you have pre-existing references to DOM elements, you can “wrap” them with jQuery's functionality.

var myInput = document.getElementById("someInput");
$(myInput);

// You can in fact do this, but you could also use a CSS Selector
$(document.getElementsByTagName("div"));

You can also fetch actual DOM elements from a jQuery Object

// Using Array Notation!             
$("#foo")[0]

// Using jQuery's .get method
$("#foo").get(0);

// Get the actual array of elements
$("#foo").get();

Pass it an HTML String

You can create new DOM elements by passing a well-formed HTML string to jQuery

// Create a new element
var newElem = $('<input id="new-element" class="foo" name="bar" />');

// Add it into to the DOM
newElem.appendTo(document.body);

You can use a similar syntax to build new elements if you don't want to write out as much HTML by passing an Object of attributes.

// Create a new element
var newElem = $('<div>',{
  id: "new-element",
  "class": "foo",
  "name": "bar"
});

Reducing Object Selections

            

            

                

                    jQuery provides methods to reduce selections with a relatively intuitive set of filters.
                

                
$("li").has("a").append('<span class="remove">x</span>');

                

                    Here we are going to append a remove span to every <li> that has an anchor tag.
                

                
$("select option").is(":selected");

                

                    Using the very common .is() filter, this checks if an option of a select is selected. You can see an interactive example of this at jsfiddle.net/CMXpl
                

            

        

        
        

            

                
Reducing Object Selections (cont.)

            

            

                

                    Aside from the mentioned examples, the jQuery API contains a full list of available methods along with extensive documentation on their use.
                

                
$("select option").not(":selected")

                

                    Filter out elements that match the selector. In this case all of the unselected options.
                

                
$("select option").map(function () { return this.value; });

                

                    Will map the selected elements to a new selection containing the return of the function. In this case you will have an array of all the <option>'s values.

Avoiding High Level Code

            

            

                

                    There are a few high-level functions that should be avoided if possible. To mention a few look at foreach, filter, map, and other functions. These functions are great, but very inefficient if you are looking for optimal performance. For comparison a raw for loop will do 3,000 iterations while a foreach will do 600. 
                

                
                    var arr = [1, 2, 3, 4, 5, 6, 7, 8, 9];
                    foreach(item in arr) { } // Slow
                    for(var i = 0, l = arr.length; i < l; i++) { } // Fast
                

            

        

        
        

            

                
Data Structure mistakes

            

            

                

                    Because JavaScript is run in-browser it is not subjected to the normal space allotments standard programs have. Memory usage is something you will want to keep an eye on as well as code performance. 
                

                

                    Array manipulations can often be improved by making sure to let the garbage collector do it's job and use arrays efficiently. Using slice to remove large chunks, or unshift for quick and native array manipulating.
